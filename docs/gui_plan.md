# gui plan

Status: **Phase 1 + Phase 2 shipped** — layout viewer with side-by-side schematic pane, cross-probing, and `E` / `Ctrl+E` hierarchy navigation. See [`gui`](/cicpy/gui) for usage. Phase 3 (placer integration via `cicpy.groups`) is next.

## Goal

Replace the C++ `cic-gui` (in `ciccreator/cic-gui/`) with a Python/PySide6 viewer hosted in `cicpy`. Add a schematic pane that renders XSchem `.sch` files alongside the `.cic` layout, with cross-probing and group-aware highlighting. Eventually drive layout placement decisions (current mirror, diff pair, cascode, stack) from naming conventions on the schematic instances.

The GUI is a **viewer**, not an editor. XSchem remains the schematic editor; ciccreator/cicpy remain the layout generators.

## Findings from current state

- **`cic-gui` (C++)** — `QWidget` + manual `paintEvent`, `CellPainter` walks the cell tree. Left pane lists cells and layers (toggleable). Wheel zoom/pan, rubber-band zoom, Y-flip via `painter.scale(z, -z)`. Reads the same `.cic` JSON cicpy already parses. Reference for feature parity.
- **`cicpy`** — full Python data model in `src/cicpy/core/`. `printer/svgprinter.py` (`DesignPrinter.printChildren`) is the natural blueprint for a Qt scene walker. No Qt code yet. Top-level `cicpy/xschem/` holds only `.sym` library files.
- **XSchem parser exists** at `src/cicpy/eda/xschem.py` (`XSchem`, `Schematic`, `Symbol`, `Component`). `Component` parses fully; geometry classes (`Line`, `Rect`, `Polygon`, `Arc`, `Text`, `Wire`) are stubs that tokenise but don't extract numeric fields. Must be extended.
- **Qt binding** — `cicsim/cmdwave_pg.py` uses **PySide6**. Match it.
- **Embedded symbol blocks (`[…]`)** — parser raises on these. Ignored for v1.

## Decisions

### Grouping data model

A `Group(kind, gid, role, members, constraints)` dataclass lives in `src/cicpy/groups.py`. Both GUI (highlight, cross-probing) and placer (layout) call `groups.resolve(cell)` for a merged list, ensuring a single source of truth.

**Three sources, in order of precedence:**

1. **Sidecar `<cell>.groups.yaml`** next to the `.cic` — explicit, machine-editable, highest priority. Used for cases where the pycell shouldn't or doesn't capture the constraint.
2. **Pycell-declared** — `<cell>.py` exposes either a `GROUPS = [...]` constant or registers via a new `addGroup(kind, gid, members, **constraints)` helper. Existing `addStack(...)` calls are introspected as implicit groups (so legacy pycells light up automatically).
3. **Name-parsing fallback** — instance-name convention for anything not covered above:

   ```
   x<kind>[_<gid>]_<role>
   ```

   - `x` — xschem prefix, ignored
   - `kind` — fixed vocabulary: `mirr | diff | casc | stack | xcpl` (final list TBD)
   - `gid` — optional group id, distinguishes multiple groups of same kind in one cell
   - `role` — `1..N`, `p/n`, `in/out`, depending on `kind`

   Examples: `xmirr_a1`/`xmirr_a2` (current mirror `a`), `xdiff_p`/`xdiff_n` (diff pair), `xcasc_in`/`xcasc_out` (cascode).

This three-tier model means highlight in the GUI always matches what the placer actually does — purely name-based grouping would lie whenever the pycell overrides via `addStack`.

### Hierarchy navigation

- click — select
- `e` — descend into hovered/selected instance (load child `.sch` and `.cic` cell)
- `Ctrl+E` — ascend
- `F` — fit current cell, `Shift+R` — reload

### Persistence model

- Annotations live in instance names → no extra files written.
- The `.cic` is regenerated by ciccreator and is **not** edited by the GUI.
- The `.sch` is owned by xschem; GUI does **not** write to it in v1. (A future "rename group" refactor could, but it's a separate, scary feature.)

## Module layout

```
src/cicpy/gui/
  app.py            QApplication bootstrap; called from CLI
  mainwindow.py     QMainWindow + QSplitter [cells/layers | layout | schem]
  layout_scene.py   Walks Design → QGraphicsItems
  layout_view.py    QGraphicsView, Y-flip, wheel/rubber-band/F-fit
  schem_scene.py    Walks Schematic + recursively renders Symbols
  schem_view.py     QGraphicsView for xschem
  sym_loader.py     Symbol path resolution + cache
  style.py          Rules → QPen/QBrush; layer visibility
  selection.py      Selection model; emits signals to both scenes

src/cicpy/groups.py   Group dataclass + resolver (yaml > pycell > names);
                      shared by GUI and placer
```

CLI: add `cicpy gui FILE.cic [--tech tech.json] [--sch FILE.sch]` to `src/cicpy/cic.py` (Click subcommand). Auto-discover tech from `<ipdir>/tech/tech.json` if `--tech` not given. If no `--sch` and no sibling `.sch`, run in layout-only mode (schem pane hidden).

PySide6 behind a `[gui]` extra in `pyproject.toml` so non-GUI users aren't forced to install Qt.

Use `QGraphicsView`/`QGraphicsScene` (not the cic-gui `paintEvent` style) — selection, hit-testing, rubber band, per-item visibility come for free, which matters for cross-probing.

## Phased plan

### Phase 1 — Layout viewer (cic-gui parity) — DONE

Mirrors `DesignPrinter.printChildren`: instances → translated/rotated `QGraphicsItemGroup`, rects → `QGraphicsRectItem` styled via `Rules`, ports → rect + label. Layer toggle iterates scene items tagged via `setData(0, layer_name)`.

**Final UX bindings** (differ slightly from initial plan):
- `F` = fit, `Z` = zoom in, `Ctrl/Cmd+Z` = zoom out, right-drag = zoom-to-area, wheel/arrows = pan, `Ctrl/Cmd+wheel` = zoom, `Shift+wheel` = pan horizontally.
- `T` = toggle all layers; per-layer toggle is a checkbox in the layer list (no theme-violating background fill).
- `Shift+R` = manual reload; `QFileSystemWatcher` also auto-reloads on disk change.
- Auto-discovery: tech file from `<ipdir>/tech/cic/*.tech`; dependency libraries from the IP's `config.yaml` (cicconf dependency keys → glob `<workspace>/<dep>/design/*.cic`). `--no-auto-libs` disables.
- `QSettings` persists window geometry, splitter sizes, last cell, and per-layer visibility (keyed by tech name).

Files shipped: `src/cicpy/gui/{app,mainwindow,layout_scene,layout_view,style}.py`. Edited `src/cicpy/cic.py` (added `gui` Click subcommand) and `pyproject.toml` (`[gui]` extra → PySide6).

### Phase 2 — Schematic pane + cross-probing — DONE

1. **Extend `src/cicpy/eda/xschem.py`** — implement `parse()` for the geometry stubs:
   - `L color x1 y1 x2 y2 {props}`
   - `B color x1 y1 x2 y2 {props}`
   - `P color n x1 y1 ... xn yn {props}`
   - `A color x y r start sweep {props}`
   - `T {text} x y rot flip xs ys {props}`
   - `N x1 y1 x2 y2 {lab=...}`
2. `gui/sym_loader.py` — resolve symref against env `XSCHEM_LIBRARY_PATH`, schematic dir, `cicpy/xschem/`, IP design dir; cache parses by absolute path.
3. `gui/schem_scene.py` — walk `Schematic.children`, instantiate components as transformed `QGraphicsItemGroup`s containing rendered symbol children. Set `item.setData(0, comp.name())` and `item.setData(1, comp)` for picking.
4. `gui/schem_view.py` — same wheel/pan/zoom/fit as layout view; no Y-flip (xschem grows Y down — verify).
5. **Cross-probing**: `gui/selection.py` `SchematicSelection` model. Click a schem component → both scenes highlight all instances whose name belongs to the same `(kind, gid)` group (parsed by `gui/groups.py`). Layout pane pans/centers on first matching instance.
6. `e`/`Ctrl+E` hierarchy nav: load child `<basename>.sch` and child cell from the same `Design`.

Files created: `gui/sym_loader.py`, `gui/schem_scene.py`, `gui/schem_view.py`, `gui/selection.py`, `gui/groups.py`. Edit `eda/xschem.py`, `gui/mainwindow.py`, `gui/layout_scene.py` (highlight wiring).

Acceptance: `cicpy gui LELOTEMP_CMP.cic --sch LELOTEMP_CMP.sch` shows readable schematic; clicking `xdiff_p` highlights `xdiff_p` and `xdiff_n` in both panes; `e` on a sub-instance descends.

### Phase 2.5 — Net highlighting

Click a wire → highlight all wires on the same `lab=` net in schem, plus all `Route` objects on that net in the layout. Uses xschem's wire `lab=` and cicpy's `Route` net field.

### Phase 3 — Placer integration

- `cicpy/groups.py` is the canonical group resolver; placer calls it to emit `addStack(...)` / `addPair(...)` calls based on the merged group list, instead of cell-specific hand-listing.
- New `addGroup(kind, gid, members, **constraints)` pycell helper for explicit declaration.
- Existing `addStack(...)` calls are introspected as implicit groups for back-compat.
- Validate that ciccreator's instance names round-trip from `.sch` → `.cic` unchanged on real designs.

## Confirmed UX/behaviour decisions

- **Tech file** — auto-discovered from `<ipdir>/tech/tech.json`; `--tech` overrides.
- **Layout-only mode** — automatic when no `.sch` is found.
- **Reload** — `QFileSystemWatcher` auto-reload on `.sch`/`.cic` save; Shift+R for manual.
- **Window state** — `QSettings` persists splitter sizes, last cell, layer visibility per tech.
- **Status bar** — coordinate readout under cursor (Å for layout, grid units for schem).
- **Theming** — dark default; theme switchable via menu.
- **Testing** — manual smoke runs only; no PNG snapshot infra in v1.

## Open items still to decide

1. **Group `kind` vocabulary** — finalize the set: `mirr | diff | casc | stack | xcpl | …` and what `role` means for each.
2. **`addGroup` helper signature** — what `constraints` keys are valid (`symmetric`, `axis`, `m`, `dummy`, …)?
3. **Sidecar `<cell>.groups.yaml` schema** — match the `Group` dataclass 1:1.
